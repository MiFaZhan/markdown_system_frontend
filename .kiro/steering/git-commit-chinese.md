# Git提交信息中文生成规则

当生成Git提交信息时，请遵循以下规则：

## 语言要求

- 所有提交信息必须使用中文
- 使用简洁、清晰的中文描述

## 提交信息格式

使用以下格式：

```
<类型>: <简短描述>

<详细说明>（可选）
```

## 提交类型（中文）

- `新增`: 添加新功能
- `修复`: 修复bug
- `更新`: 更新现有功能
- `重构`: 代码重构
- `样式`: 样式调整（不影响功能）
- `文档`: 文档更新
- `测试`: 添加或修改测试
- `构建`: 构建系统或依赖更新
- `配置`: 配置文件更改

## 示例

- `新增: 添加用户登录功能`
- `修复: 解决页面加载缓慢问题`
- `更新: 优化项目列表显示效果`
- `重构: 重构用户认证模块`

请确保提交信息简洁明了，准确描述代码变更内容。

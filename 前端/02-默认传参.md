# JavaScript 语法整理

## 1. 导出函数 `export function`

```javascript
export function getProjectList(params = {}) { ... }
```

- `export`：导出函数，让其他文件可通过 `import` 引入
- `params = {}`：参数默认值，如果不传参数，`params` 默认为空对象

---

## 2. 对象展开运算符 `{ ...obj1, ...obj2 }`

合并两个对象，**右侧属性覆盖左侧同名属性**：

```javascript
const a = { pageNum: 1, pageSize: 10 }
const b = { pageSize: 20, sortOrder: 'desc' }

const merged = { ...a, ...b }
// 结果: { pageNum: 1, pageSize: 20, sortOrder: 'desc' }
```

所以调用 `getProjectList({ pageSize: 20 })` 会变成：
```javascript
{ pageNum: 1, pageSize: 20, sortField: 'creation_time', sortOrder: 'asc' }
```

---

## 3. JSDoc 注释

```javascript
/**
 * 分页查询项目列表
 * @param {Object} params - 查询参数
 * @param {number} params.pageNum - 页码，默认1
 * @param {number} params.pageSize - 每页大小，默认10
 */
```

- 文档注释，IDE 可识别并提供代码提示
- `@param`：描述参数
- `@returns`：描述返回值

---

## 4. 为什么默认参数要分开写？

```javascript
export function getProjectList(params = {}) {
  const defaultParams = {
    pageNum: 1,
    pageSize: 10,
    sortField: 'creation_time',
    sortOrder: 'asc'
  }
  return get('/project', { ...defaultParams, ...params })
}
```

| 写法 | 特点 |
|------|------|
| 分开写 | 默认参数一目了然，修改时只改一处 |
| 写进 `= {}` | 更简洁，但参数多时函数签名很长 |

**必须分开写的情况**：默认值需要动态计算时

```javascript
const defaultParams = {
  pageSize: getConfig('pageSize') || 10,  // 动态获取
  creatorId: getCurrentUserId()            // 运行时计算
}
```

## 语法速查表

| 语法 | 作用 |
|------|------|
| `export function` | 导出函数供其他文件使用 |
| `params = {}` | 参数默认值 |
| `{ ...a, ...b }` | 合并对象，右侧覆盖左侧 |
| `/** */` | JSDoc 文档注释 |
